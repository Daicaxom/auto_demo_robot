1. Giới Thiệu Test Spec

Test Spec (Specification Layer) là phần định nghĩa các test case theo cú pháp Gherkin. Mục tiêu của Spec Layer là cung cấp các test case dễ đọc, thân thiện với các thành viên không chuyên về lập trình, đồng thời phân tách rõ ràng giữa mô tả nghiệp vụ và thực thi của automation.

2. Nguyên Tắc Viết Test Spec

  • Đọc dễ hiểu: Sử dụng cú pháp Gherkin (Given/When/Then) để mô tả quy trình test.
  • Tách biệt trách nhiệm: Spec chỉ định nghĩa test case, gọi các luồng đã được xây dựng ở Flow Layer.
  • Nhất quán: Giữ kiểu viết và định dạng ổn định giữa các test spec để dễ bảo trì.

3. Tăng Tốc Viết Test Spec

Để tạo Test Spec nhanh chóng, bạn có thể:

  • Sử dụng Templates: Tạo file mẫu cho test spec với cấu trúc Gherkin và các thiết lập chuẩn, sau đó copy để tái sử dụng.
  • Sử dụng Snippets: Lập trình viên có thể thiết lập snippets trong IDE (VS Code, PyCharm) để tự động chèn các block code mẫu.
  • Reuse Resources: Import các file Flow và POM đã được xây dựng để Spec chỉ cần tham chiếu thay vì viết lại luồng thao tác.

4. Template Test Spec

Dưới đây là một mẫu Test Spec theo cú pháp Gherkin mà bạn có thể điều chỉnh lại tùy theo ứng dụng của mình: 

*** Settings ***
Resource          ../Resources/TestFlows/ExampleFlow.robot
Test Setup        Example Flow Setup
Test Teardown     Close Browser

*** Test Cases ***
User Can Perform Basic Operation
    [Tags]    smoke    regression
    Given Application is ready for input
    When User performs the example operation with input "value1" and "value2"
    Then Result should be "ExpectedResult"

5. Best Practices Khi Viết Test Spec

  • Rõ ràng và Ngắn gọn: Tránh lặp lại mô tả đã có trong Flow Layer. Test Spec chỉ nên tập trung vào định nghĩa các bước nghiệp vụ.
  • Gắn Thẻ (Tags): Sử dụng tags để phân loại test case (ví dụ: smoke, regression, critical) nhằm chạy test theo mục đích.
  • Import Chính Xác: Đảm bảo các file Flow và Resource cần thiết được import đúng, giúp test spec thực thi liền mạch.
  • Consistent Language: Duy trì ngôn ngữ thống nhất trong toàn bộ Spec Layer để dễ hiểu và bảo trì.
  • Document: Mỗi test case nên có một mô tả ngắn gọn, giải thích ý nghĩa của test cho mọi thành viên trong team.

6. Kết Luận

Việc xây dựng Test Spec bằng cách sử dụng cú pháp Gherkin giúp cho các test case trở nên dễ đọc, thân thiện và dễ bảo trì. Sử dụng template và snippets là cách hiệu quả để tăng tốc quá trình viết test case và đảm bảo tính nhất quán trong toàn bộ dự án.

Lưu ý: Khi có sự thay đổi về nghiệp vụ hay luồng test, chỉ cần cập nhật ở Flow Layer, Test Spec sẽ tự động cập nhật thông qua resources đã import.
